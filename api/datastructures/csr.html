class pigo::CSR¶

Holds compressed sparse row matrices or graphs.

This is a fundamental object in PIGO. It is used to represent sparse matrices and graphs. It can be loaded directly from files that are in adjacency formats, such as CHACO/METIS files.

In many cases, this is the desired format for a graph or matrix in memory. When sparse graphs or matrices are delivered in COO formats, such as matrix market or edge lists, they are frequently converted to CSR. This class can automatically handle such conversions internally.

Template Parameters

  • Label: the label data type. This type needs to be able to support the largest value read inside of the COO. In a graph this is the largest vertex ID.

  • Ordinal: the ordinal data type. This type needs to support large enough values to hold the number of endpoints or rows in the COO. It defaults to the same type as the label type.

  • LabelStorage: the storage type of the endpoints of the CSR. This can either be vector (std::vector<Label>), a pointer (Label*), or a shared_ptr (std::shared_ptr<Label>).

  • OrdinalStorage: the storage type of the offsets of the CSR. This can either be vector (std::vector<Ordinal>), a pointer (Ordinal*), or a shared_ptr (std::shared_ptr<Ordinal>).

  • weighted: if true, support and use weights

  • Weight: the weight data type. This type needs to be able to support the largest value read inside of the COO. In a graph this is the largest vertex ID.

  • WeightStorage: the storage type for the weights. This can be a raw pointer (Weight*), a std::vector (std::vector<Weight>), or a std::shared_ptr<Weight>.

template<class COOLabel, class COOOrdinal, class COOStorage, bool COOsym, bool COOut, bool COOsl, bool COOme, class COOW, class COOWS>
CSR(COO<COOLabel, COOOrdinal, COOStorage, COOsym, COOut, COOsl, COOme, weighted, COOW, COOWS> &coo)¶

Initialize from a COO.

This creates a CSR from an already-loaded COO.

Note that this will densely fill in all labels, so if there are many empty rows there will be unnecessary space used.

Template Parameters

  • COOLabel: the label for the COO format

  • COOOrdinal: the ordinal for the COO format

  • COOStorage: the storage format of the COO

  • COOsym: whether the COO is symmetrized

  • COOut: whether the COO only keeps the upper triangle

  • COOsl: whether the COO removes self loops

  • COOme: whether the COO removes multiple edges

  • COOW: the weight type of the COO

  • COOWS: the weight storage type of the COO

Parameters

  • coo: the COO object to load the CSR from

Utility to free consumed memory.
