class pigo::BaseGraph : public pigo::CSR<uint32_t, uint32_t, uint32_t*, uint32_t*, false, float, float*>¶

Used to load graphs from disk.

This class provides a graph-specific naming on top of CSRs.

Template Parameters

  • vertex_t: the type to use for vertices

  • edge_ctr_t: the type to use for an edge counter

  • edge_storage: the storage type of the endpoints of the CSR.

  • edge_ctr_storage: the storage type of the offsets of the CSR.

  • weighted: if true, support and use weights

  • Weight: the weight data type.

  • WeightStorage: the storage type for the weights. This can be a raw pointer (Weight*), a std::vector (std::vector<Weight>), or a std::shared_ptr<Weight>.

class pigo::DiGraph¶

Used to load directed graphs from disk.

This class provides a graph-specific naming on top of non-symmetric matrices (Matrix).

Template Parameters

  • vertex_t: the type to use for vertices

  • edge_ctr_t: the type to use for an edge counter

  • edge_storage: the storage type of the endpoints of the CSR/CSC.

  • edge_ctr_storage: the storage type of the offsets of the CSR/CSC.

  • weighted: if true, support and use weights

  • Weight: the weight data type.

  • WeightStorage: the storage type for the weights. This can be a raw pointer (Weight*), a std::vector (std::vector<Weight>), or a std::shared_ptr<Weight>.

Public Functions

template<class COOvertex_t, class COOedge_ctr_t, class COOStorage, bool COOsym, bool COOut, bool COOsl, bool COOme, class COOW, class COOWS>
inline DiGraph(COO<COOvertex_t, COOedge_ctr_t, COOStorage, COOsym, COOut, COOsl, COOme, weighted, COOW, COOWS> &coo)¶

Initialize from a COO.

This creates a DiGraph from an already-loaded COO.

This first copies the COO and transposes the copy, setting that as the in-edges. The out-edges are the original COO.

Template Parameters

  • COOvertex_t: the label for the COO format

  • COOedge_ctr_t: the ordinal for the COO format

  • COOStorage: the storage format of the COO

  • COOsym: whether the COO is symmetrized

  • COOut: whether the COO only keeps the upper triangle

  • COOsl: whether the COO removes self loops

  • COOme: whether the COO removes multiple edges

  • COOW: the weight type of the COO

  • COOWS: the weight storage type of the COO

Parameters

  • coo: the COO object to load the CSR from
